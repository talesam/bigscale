# Security

## Architecture overview

BigScale's admin panel never exposes the server API key to the browser:

- The `BIGSCALE_API_KEY` is read **server-side** from a file mounted on the panel container (`/shared/api-key`, generated by the `bigscale-init` service on first start).
- All browser calls hit `/api/bs/v1/*`, a proxy route that validates the session cookie and adds the `Authorization: Bearer …` header server-side before forwarding to the BigScale server.
- The session cookie (`bs_session`) is `httpOnly`, `sameSite=strict`, and stored in-memory on the server.
- The admin password is hashed with `scrypt` (N=16384, r=8, p=1) and stored in `/data/admin.json` inside the panel container's persistent volume. The default `admin` / `bigscale` credentials force a password change on first login before any other route is reachable.

## Reporting issues

For vulnerabilities affecting the panel, the proxy, or the bootstrap flow, open a private security advisory on GitHub or contact the maintainer directly.

For issues in the upstream coordinator engine, please report them at https://github.com/juanfont/headscale.
